# Design Document


## Architecture
- Three-tier MERN architecture: React frontend, Express/Node backend, and MongoDB database.
- Client communicates with REST APIs over HTTPS; backend enforces JWT authorization.

## UML Use Cases
- User: Register/Login, calculate BMI/BMR, view calorie needs, manage profile, track nutrition/water, set goals, view progress.
- Admin: Login, monitor users, manage accounts, review activity metrics.

## Activity Diagram Description
- User authenticates, enters health inputs, receives nutrition recommendations, logs daily intake, and views dashboard trends for decision support.

## Class Diagram Description
- User class stores identity, role, profile, and goals.
- MealPlan class stores recommendation metadata, macro targets, and assignment details.
- Progress class stores date-wise calories, macros, water intake, and weight updates.
- AuthToken and middleware components enforce secure access control.

## Database Design
- Collections: users, mealplans, progress.
- users links to assigned plans and preferences.
- progress stores time-series entries for dashboard analytics.

## API Design
- Auth APIs for registration and login.
- User APIs for profile management and dashboard data retrieval.
- Nutrition APIs for calculators, food data, and meal recommendations.
- Progress APIs for logging and fetching nutrition/water/weight history.
- Admin APIs for platform-level user and access management.
